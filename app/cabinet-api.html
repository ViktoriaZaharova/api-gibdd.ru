Документация API сервиса API-GIBDD.RU

Авторизация

API интерфейс используется для того, чтобы интегрировать возможности сервиса в личный проект клиента. API предназначен для разработчиков и сопровождается детальной документацией.

Описание

API сервиса API-GIBDD работает по протоколу HTTP и представляет собой набор методов, с помощью которых совершаются запросы и возвращаются ответы для каждой операции. Все ответы приходят в виде JSON структур.

Авторизация

Процесс авторизации возможен только при наличии персонального ключа (API-key), которым в дальнейшем сопровождается каждый запрос к API. Необходимый ключ можно получить в личном кабинете во вкладке API.

Формат входящего запроса

API сервиса позволяет получать данные, принимая разные варианты запросов, если у Вас уже реализовано обращение к стороннему API, Вы можете легко и без особых изменений получать информацию посредством нашего, изменения коснуться только домена и токена. В примере ниже будет рассмотрен формат, с которым работает сам официальный сервис ГИБДД. Ниже в пример API-key передается в параметре reCaptchaToken.

Запрос списка последних заявок

Для того, чтобы получить список последних заявок необходимо отправить запрос с обязательными параметрами.

Обязательные параметры:

countday количество дней (не более 14)

Пример запроса:

https://05kbm.ru/lka/api/v1/YOUR_API_KEY/tickets/getlist/?countday=7

Пример ответа при успешном создании заявки:

    {
        status: true,
        data: {
        {
            ticketId: 3687,
            dateCreate: 2019-11-17 15:32:29,
            fio: Иванов Сергей Петрович,
            kbmWhenCreating: 1.0,
            status: new
        },{
            ticketId: 3651,
            dateCreate: 2019-08-17 13:02:38,
            dateClose: 2019-08-18
            fio: Иванова Елена Павловна,
            kbmWhenCreating: 1.1,
            kbmWhenClose: 0.8,
            status: done
        }
    }

Возможные ошибки:

030 Missing countday parameter / Отсутствует обязательный параметр countday
031 Invalid countday parameter / Некорректный параметр countday
032 The parameter countday must not exceed 14 days / Параметр countday не должен превышать 14

Демо-режим

Дополнительно имеется так называемый демо-режим, на котором Вы можете протестировать запросы и ответы на них не создавая реальных заявок.

Для того, чтобы воспользоваться демо-режимом, достаточно в ссылке запроса изменить **api** на **demo-api**.

Например ,чтобы получить актуальную инфомацию об аккаунте в режиме demo:

https://05kbm.ru/lka/demo-api/v1/YOUR_API_KEY/account/info

Запрос истории водительских прав

Получить историю водительских прав можно выполнив запрос:

Обязательные параметры:

fio ФИО водителя
driverLicense Серия и номер водительского удостоверения

Пример запроса:

https://05kbm.ru/lka/api/v1/YOUR_API_KEY/history_vu/create/?fio=Иванов Иван Иванович&driverLicense=1111 321321

Пример ответа:

    {
        status: true,
        data: {
            requestId: 352
        }
    }

020 Missing driverLicense parameter / Отсутствует обязательный параметр - driverLicense.
